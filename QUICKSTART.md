# Quick Start Guide

## 5-Minute Setup

### 1. Create GitHub Repository (2 minutes)
- Go to github.com → New repository
- Name: `YOUR_USERNAME.github.io` (e.g., `emilresearcher.github.io`)
- Set to Public → Create

### 2. Upload Files (2 minutes)
- Click "uploading an existing file"
- Upload: `index.html`, `style.css`, `profile.jpg`, `cv.pdf`
- Commit changes

### 3. Enable GitHub Pages (1 minute)
- Go to Settings → Pages
- Source: Branch = main, Folder = / (root)
- Save

### 4. Done!
Visit: `https://YOUR_USERNAME.github.io/`

## Priority Customizations

Before going live, update in `index.html`:

1. Line 10: `<title>Emil - Researcher</title>` → Your name
2. Line 16: `<div class="nav-brand">Emil</div>` → Your name
3. Line 27-32: Your name, title, affiliation, and contact info
4. Lines 41-51: Your bio
5. Lines 86-126: Your publications
6. Lines 132-155: Your education and CV details

## What Each File Does

- **index.html** - Main website content (edit your information here)
- **style.css** - Styling and colors (change colors here if desired)
- **profile.jpg** - Your photo (upload a square photo)
- **cv.pdf** - Your CV (optional, for download button)
- **README.md** - Full documentation (for your reference)

## Make It Yours

Search and replace in `index.html`:
- "Your Name" → Your actual name
- "your.email@illinois.edu" → Your email
- "YOUR_PROFILE" → Your Google Scholar ID
- "YOUR_USERNAME" → Your GitHub username

## Tips

- Use a 400x400px (or larger) square photo for best results
- Keep descriptions concise and impactful
- Update regularly with new publications
- Test on mobile after deploying
